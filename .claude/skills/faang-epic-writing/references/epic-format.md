# Epic Document Format

Strict template for every epic document generated by this skill. Every section is mandatory unless marked optional.

## Template

```markdown
# E{phase}.{sequence}: {Imperative Title}

**Phase:** {N} - {Phase Name}
**Class:** {Infrastructure | Feature | Integration}
**Design Doc Reference:** {section numbers from the design doc}
**Dependencies:**

- Phase {N}: {Phase name} (exit criteria met)
- E{N}.{M}: {Epic title} (specific deliverable required)
- Asset: {asset name} (available in asset catalog)
- Service: {ServiceName} interface (public API available)

---

## Goal

{Single sentence. Imperative mood. Describes the buildable outcome, not the process.}

---

## Scope

### File Inventory

| File                   | Action  | Responsibility |
| ---------------------- | ------- | -------------- | -------------------------------------------------- |
| `{path/to/file.swift}` | {Create | Modify}        | {Single sentence: what this epic does to the file} |

### Integration Points

**{FileName.swift}**

- Imports from: {list of project files this file imports}
- Imported by: {list of project files that import this file}
- State reads: {AppState properties read, or "None"}
- State writes: {AppState properties mutated, or "None"}

{Repeat for each file in the inventory.}
```

## Field Definitions

### Header Fields

| Field                | Requirement | Content                                                               |
| -------------------- | ----------- | --------------------------------------------------------------------- |
| Epic ID              | Mandatory   | `E{phase}.{sequence}` format, matching PHASES.md                      |
| Title                | Mandatory   | Imperative verb + scope. Under 60 characters                          |
| Phase                | Mandatory   | Phase number and name from PHASES.md                                  |
| Class                | Mandatory   | Exactly one of: Infrastructure, Feature, Integration                  |
| Design Doc Reference | Mandatory   | Section numbers (e.g., "SS2, SS4.2, SS7.3") from the design document  |
| Dependencies         | Mandatory   | Explicit list. Empty dependencies state "None" (rare for non-Phase-0) |

### Goal

- Exactly one sentence
- Imperative mood ("Build", "Implement", "Define", "Wire")
- Describes the deliverable, not the process
- No hedging ("try to", "attempt to", "explore")

### File Inventory

- Every file the epic creates or modifies, listed exhaustively
- Action column is exactly `Create` or `Modify`
- Responsibility column is exactly one sentence
- Files not listed are not in scope

### Integration Points

- Required for every file in the inventory
- Reveals hidden coupling before implementation begins
- "None" is a valid value for any field

### Out of Scope

- Minimum 3 items
- Must include at least one item from each category: adjacent features, later-phase work, deferred polish
- Items must be specific enough that an engineer would recognize them as potentially in scope

### Definition of Done

- Minimum count per complexity tier (see SKILL.md SS4.4)
- Every item is binary verifiable
- Every item uses present tense
- Every item describes an observable outcome, not a development step
- Items ordered by lifecycle: initialization, interaction, state, teardown, edge cases

### Implementation Notes

- Optional section, but preferred for non-trivial epics
- Only concrete guidance with citations
- No speculative suggestions
- No alternative approaches ("you could also...")
- Reference design doc sections, token names, API contracts, or architectural patterns

## Validation Checklist

Before finalizing an epic document, verify:

- [ ] Epic ID matches PHASES.md entry
- [ ] Title is imperative mood, under 60 characters
- [ ] Goal is exactly one sentence
- [ ] Every dependency is resolvable in PHASES.md
- [ ] File inventory lists every file with Action and Responsibility
- [ ] Integration points documented for every file
- [ ] Out of Scope contains minimum 3 specific exclusions
- [ ] Every DoD item is binary verifiable (YES/NO answer)
- [ ] No prohibited phrases appear in DoD items
- [ ] DoD meets minimum item count for the epic's complexity tier
- [ ] DoD covers applicable lifecycle categories
- [ ] Implementation notes cite sources (no unsourced speculation)
- [ ] Output file path follows naming convention
